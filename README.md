# 프로세스 설계 (process_designer)

조선소 시운전팀의 업무를 **lv0~lv6 계층 카드**로 그리고, 각 업무에 **AI 에이전트를 적용했는지 /
어떤 기술을 쓰는지 / 어느 부서가 맡는지**를 기록하는 Streamlit 앱입니다. 공용PC에 띄워두고
여러 명이 함께 편집합니다.

```
lv0 조선 › lv1 생산 › lv2 시운전     ← 고정
        └ lv3 부문      (선장운전 / 전장운전 / 기장운전 / 기획운영 / 공통업무 /
                         해운부 / 코멘더 / 해양-유틸·프로세스 / 해양-안전 / 해양-전계장 … 추가 가능)
            └ lv4 대분류 └ lv5 중분류 └ lv6 세부업무   ← 자유 추가
```

## 두 가지 버전

| | 메인앱 | **개인 작성용 (standalone)** |
|---|---|---|
| 누가 | 관리자 — 공용PC 한 대 | 팀원 각자 |
| 무엇 | 전체 트리 관리 · 개인 파일 취합 · 엑셀 | 자기 업무만 작성 → 제출 파일 만들기 |
| 실행 | `run.bat` → http://localhost:8540 | **HTML 파일 하나 더블클릭** (서버·설치 불필요) |

### 개인 작성용 HTML 받기

**빌드된 파일이 저장소에 함께 들어 있습니다** — 바로 받아서 쓰면 됩니다.

```
dist/프로세스설계_개인작성용_r3.html      ← 이 파일 하나를 팀원에게 전달
```
GitHub 에서 받을 땐 그 파일을 열고 **[Download raw file]** 을 누르세요.
받은 사람은 더블클릭만 하면 됩니다 — 설치도, 인터넷도 필요 없습니다.

**소스를 고쳤다면 다시 구우세요:**
```bat
python build_standalone.py
```
> ⚠ `frontend/index.html` 을 고치고 이걸 안 돌리면 **배포판이 옛 버전인 채로 남습니다.**
> 파일명은 고정이라 재빌드하면 덮어써지고, `dist/` 도 함께 커밋해야 합니다.

### 과별 기본 골격(lv3~lv5) 탑재 HTML 배포

각 과에 **자기 골격이 미리 깔린 HTML**을 나눠주려면 골격 JSON 을 시드로 굽습니다. 열면 그 과의
lv3~lv5 가 이미 있고 소속(과)도 자동 지정돼, 세부업무(lv6)만 채워 제출하면 됩니다.

```bat
python build_standalone.py --seed 골격.json           REM 과 1개 → dist/프로세스설계_<과>.html
python build_standalone.py --seeds <골격폴더>          REM 폴더의 프로세스_*.json 마다 과별 HTML 일괄
```
- 골격 JSON = 이 앱/개인배포판이 **내보낸 형식**(`exported_dept` 에 과가 들어 있으면 파일명·소속 자동).
- **받은 과 JSON 을 다듬어** 골격으로 쓸 때는 표준 HTML 의 *이어서 작성하기* 로 열어 정리→내보내기.
  (불러오기는 화면을 **통째로 덮어씁니다**·되돌리기 없음 → **원본 백업 필수**.)
- 이미 자기 브라우저에 작성분(localStorage)이 있으면 시드는 무시하고 **이어서** 작성합니다(작업분 보호).

### 부서/과는 2단(부서 → 과)

노드에는 **과만 저장**하고 부서는 매핑(`schema.DEPT_TREE`)으로 자동 표시·집계합니다. 상단·세부업무의
부서/과 선택칸은 `<optgroup>` 으로 부서 아래 과가 뜹니다. 조직표가 바뀌면 `schema.py` 의 `DEPT_TREE`
와 `frontend/index.html` 의 `DEPT_TREE` 를 **함께**(twin) 고치세요.

## 시작하기 (메인앱)

```bat
setup_env.bat      REM 최초 1회 — venv 생성 + 패키지 설치 + 환경점검 + 실행
run.bat            REM 이후 실행
```
→ 브라우저에서 http://localhost:8540

셋업이 안 될 때는 `python check_env.py` 로 원인을 먼저 확인하세요.

## 쓰는 법

1. 상단에 **작성자 이름**을 입력합니다. (저장 기록에 남습니다)
2. **계층 편집** 탭에서 lv3 카드를 클릭 → 오른쪽 컬럼이 열립니다. `＋` 로 하위 업무를 추가합니다.
3. **드래그**로 순서를 바꾸고, 카드를 **왼쪽 컬럼의 다른 카드 위로 떨어뜨리면** 상위 업무가 바뀝니다.
4. 카드를 선택하면 오른쪽 패널에서 내용을 편집합니다. **레벨에 따라 입력 항목이 다릅니다.**
5. **변경은 상단 [💾 저장] 을 눌러야 파일에 반영**됩니다. 저장 전까지는 나만 보입니다.

### 알아두면 좋은 것

- **레벨을 잘못 만들었다면** 지우지 말고 상세 패널 맨 아래 `상위 업무 바꾸기` 를 쓰세요. 하위 업무까지
  함께 옮겨지고 레벨이 다시 매겨집니다. (드래그는 같은 레벨 안에서만 움직입니다)
- **업무명 검색** — 상단 검색창에 입력하면 맞는 카드가 노란 테두리로 표시되고 건수가 나옵니다.
- **목록에 없는 기술**은 상세 패널 기술 칩 아래 입력칸에 적고 Enter — 이 업무에 적용되면서
  도메인 목록에도 등록됩니다. (쉼표로 여러 개)
- **`▦ 컬럼`** 은 단계별로 파고드는 보기, **`☰ 전체보기`** 는 전체 계층을 들여쓰기로 펼친 보기입니다.
- 하위가 있는 업무를 삭제하거나, 저장하지 않은 채 `↻ 다시 읽기` 를 누르면 **확인 창**이 뜹니다.

### 레벨별 입력 항목

| 레벨 | 입력하는 것 |
|---|---|
| lv3 부문 · lv4 대분류 · lv5 중분류 | **이름 + 설명**만 — 업무를 묶는 분류이기 때문 |
| lv6 세부업무 | **전체** — AI 에이전트 유무, 활용 기술, 부서/과, 자동화 수준, 담당자, 수행 주기, **1회 소요시간 · 연간 횟수**, 산출물, 연계 시스템(도메인 선택 + 추가정보), 설명 |

### 발생 패턴 — 개인 작성용에서만

세부업무(lv6)마다 **언제 하는 업무인지**를 고릅니다. 이게 나중에 "연간 몇 척 × 각 호선 일정"과
곱해져 **날짜별 부하**가 됩니다.

| 패턴 | 무엇 | 적는 것 |
|---|---|---|
| **상시루틴** | 호선과 상관없이 늘 하는 일 | 기간단위 + 횟수 (예: `주 3회`) |
| **호선루틴** | 호선 도는 동안 반복하는 일 | 기간단위 + 횟수 + **반복 구간**(복수 선택) |
| **호선이벤트** | 특정 시점에만 하는 일 | 기준 시점(G/T·D/L 등) + 오프셋 + **호선당 횟수** |

- **호선 업무는 "호선당" 몇 번인지** 적습니다 (연간이 아닙니다). 예: LNG 벙커링 `호선당 2회`.
  연간은 건조 척수를 곱해 나중에 산출하므로, 척수 계획이 바뀌어도 다시 적을 필요가 없습니다.
- **선종은 대분류(lv4)에서** 고릅니다 — 여러 개 선택 가능. 그 아래 세부업무가 물려받으니
  업무마다 다시 고르지 않아도 됩니다.
- 반복 구간·선종은 **칩을 눌러 여러 개** 고를 수 있습니다. 카드를 쪼갤 필요가 없습니다.

### 작업시간 — 자동화 우선순위의 근거

lv6 에 **1회 소요시간**(시간, 30분=`0.5`)과 **연간 횟수**를 넣으면 **연간 공수**가 자동으로 계산됩니다.

```
0.5시간 × 52회 = 연간 26시간
```

**수행 주기를 고르면 연간 횟수가 자동으로 채워집니다** (주 1회→52, 일 1회→250 *근무일 기준*,
월 1회→12, 분기→4, 연 1회→1). `호선별`·`수시`는 정해진 횟수가 없어 직접 넣습니다.
직접 고친 숫자는 주기를 바꿔도 덮어쓰지 않습니다.

어떤 업무가 시간을 가장 많이 먹는지가 보이므로, **AI 에이전트를 어디부터 붙일지** 판단할 수 있습니다.

AI 에이전트를 적용하고 담당자가 붙는 실체는 **lv6 세부업무**입니다. 상위 레벨에 같은 항목을
반복 입력할 필요가 없습니다. 요약의 AI 적용률도 **lv6만 세므로** 분류 단계가 "미적용"으로
잡혀 수치가 왜곡되지 않습니다.

> lv6 업무를 lv5 로 올려도 입력해둔 AI·기술·담당자는 **지워지지 않고 보존**됩니다. 화면에서만
> 숨겨지고, 다시 lv6 으로 내리면 그대로 되살아납니다. (엑셀에는 그 값이 계속 보입니다)

### 화면 4개

| 탭 | 하는 일 |
|---|---|
| 계층 편집 | 카드 배치·드래그·검색·상세 편집 |
| 도메인 관리 | 부서/과 · 활용기술 · 자동화수준 · 수행주기 선택 목록 관리 |
| 엑셀 가져오기 / 내보내기 | 엑셀 일괄등록 · 엑셀/JSON 다운로드 · 요약 통계 |
| 이력 · 복원 | 저장 스냅샷 목록에서 원하는 시점으로 되돌리기 |

## 개인 작성용 — 팀원 안내

받은 HTML 파일을 **더블클릭**하면 열립니다. 설치할 것도, 접속할 주소도 없습니다.

1. 상단에 **소속**과 **이름**을 고르고 적습니다. (딱 한 번)
2. 내 부문 카드를 눌러 `＋` 로 대분류 → 중분류 → 세부업무를 만듭니다.
3. **세부업무(lv6)** 에만 AI 에이전트·활용기술·자동화수준·수행주기·작업시간을 넣습니다.
   - **부서·담당자는 안 물어봅니다** — 위에 적은 소속·이름이 자동으로 들어갑니다.
4. 다 되면 **[⬇ 제출 파일 내보내기]** → `.json` 파일을 관리자에게 보냅니다.

> **작성 내용은 이 브라우저에 자동 저장됩니다.** 창을 닫았다 다시 열어도 이어서 쓸 수 있습니다.
> 다만 PC 를 바꾸거나 브라우저를 초기화하면 사라지니, 끝나면 꼭 **내보내기**를 하세요.
> 내보낸 파일은 `내보내기 / 불러오기` 탭에서 다시 불러와 이어서 작성할 수 있습니다.

메인앱과 다른 점: 서버가 없으므로 **이력·복원**, **충돌 경고**, **다시 읽기** 가 없습니다
(혼자 쓰는 파일이라 필요가 없습니다). 담당자 이름도 가리지 않고 그대로 보입니다.

## 개인 파일 취합 (관리자)

팀원들이 보낸 `.json` 을 **`엑셀 가져오기 / 내보내기` → `⬆ 가져오기`** 에 하나씩 올리면 됩니다.
엑셀과 똑같이 **추가/변경/삭제 건수를 먼저 보여주고** 확인 후 반영합니다.

- **순서대로 여러 개 올려도 서로 덮어쓰지 않습니다.** 삭제 옵트인이 꺼져 있으면 파일에 없는
  남의 업무는 그대로 보존됩니다.
- **부문은 중복되지 않습니다.** 배포판과 메인앱이 같은 부문 id 를 쓰고, 각자 새로 만든
  같은 이름의 부문도 하나로 합쳐집니다.
- 같은 사람이 **다시 제출해도 복제되지 않고 갱신**됩니다.
- 개인이 임의로 만든 기술은 **바로 마스터에 섞이지 않습니다** — "도메인 목록에 없는 값이
  있습니다" 로 알려주고, 승인해야 목록에 들어갑니다.

## 엑셀 일괄등록

업무가 많을 때는 화면에서 하나씩 만들지 말고 엑셀로 채우세요.

1. **엑셀 가져오기 / 내보내기** → `⬇ 내보내기` → `엑셀 다운로드`
2. `계층도` 시트에서 lv3~lv6 칸을 채웁니다. **왼쪽부터 연속으로** 채워야 하고, 상위 업무는 반드시
   자기 행이 있어야 합니다. `id` 칸이 비어 있으면 새 업무로 등록됩니다.
   AI·기술·부서 같은 세부 정보는 **lv6 행에만** 채웁니다.
3. `⬆ 가져오기` 탭에 올리면 **추가/변경/삭제 건수와 목록을 먼저 보여줍니다.** 확인하고
   `[이 내용으로 반영]` 을 눌러야 실제로 반영됩니다.
   - 엑셀에서 행을 지워도 **기본은 삭제하지 않습니다** (실수 방지). 삭제하려면 체크박스를 켜세요.
   - 도메인 목록에 없는 값이 있으면 알려주고, 목록에 추가할지 고를 수 있습니다.
   - 담당자가 마스킹된 값(`홍*동`)이면 원본 이름을 덮어쓰지 않습니다.
4. 반영 후 상단 **[💾 저장]** 을 눌러야 파일에 기록됩니다.

## 여러 명이 같이 쓸 때

- 자유롭게 편집하고 저장하면 됩니다. 저장할 때마다 **직전 버전이 스냅샷으로 보관**됩니다.
- 내가 편집하는 동안 다른 사람이 먼저 저장했다면, 저장 시 **경고가 뜨고** 세 가지 중에 고릅니다:
  최신 내용 불러오기 / 내 것으로 덮어쓰기 / 취소.
- **덮어써도 상대 작업은 사라지지 않습니다** — 이력 화면에서 그 시점으로 복원할 수 있습니다.
- 상단에 "○○○ 님이 방금 저장했습니다" 배너가 뜨면 `↻ 다시 읽기` 로 최신본을 받으세요.
  저장하지 않은 편집이 있으면 사라진다고 먼저 알려줍니다.
- **복원 화면의 "되살아남 / 값이 바뀜 / 사라짐" 숫자는 실제 계산값**입니다. 되돌리기 전에
  사라질 업무 목록까지 확인할 수 있습니다.

## 데이터 위치

`<데이터 base>/process/process_tree.json` (사외망 기본: `code_N\data\process`)
- `.env` 의 `PROCESS_DATA_PATH` 로 위치를 바꿀 수 있습니다.
- 사내망에서는 NAS 경로 아래 `process/` 로 자동 전환됩니다.
- 스냅샷은 같은 폴더 `history/` 에 최근 90일 + 최신 50개가 보관됩니다.

## 개인정보

담당자 이름은 파일에 원본으로 저장하고, **화면 카드·표·엑셀에서는 마스킹**해서 보여줍니다
(`홍길동` → `홍*동`). 상세 편집 입력칸에만 원본이 보입니다. 엑셀 다운로드의 마스킹 체크박스를
끄면 원본이 파일에 들어가므로 외부 공유에 주의하세요.

## 화면이 안 뜰 때

이 앱은 화면 전체가 하나의 컴포넌트라, `frontend/` 파일(`index.html`·`sortable.min.js`)이나
`pyarrow` 패키지가 없으면 화면이 뜨지 않습니다. **`python check_env.py`** 가 이 둘을 검사하니
먼저 돌려보세요. `setup_env.bat` 으로 설치하면 두 조건 모두 갖춰집니다.

## 포함된 오픈소스

- [SortableJS](https://github.com/SortableJS/Sortable) 1.15.6 (MIT) — `frontend/sortable.min.js`.
  인터넷 없는 환경에서도 동작하도록 CDN 대신 파일로 포함했습니다.
